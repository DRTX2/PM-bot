\section{TÍTULO}

Desarrollo, Arquitectura e Implementación del \textbf{Bot Project Manager AI PetSafe}: Orquestación Inteligente Event-Driven, Integración de Ecosistemas de Desarrollo y Modelado de Decisiones Asistido por Inteligencia Artificial Generativa.

\section{INTRODUCCIÓN}

La gestión de proyectos de ingeniería de software modernos requiere una supervisión constante sobre repositorios de código, tableros de metodologías ágiles y plataformas de comunicación. En el contexto del proyecto PetSafe, una plataforma digital integral orientada a la gestión clínica veterinaria, el equipo de desarrollo detectó una fricción operativa significativa: la información técnica y administrativa se encontraba fragmentada entre GitHub (código fuente), Trello (gestión ágil) y Discord (comunicación de equipo). Esta separación creaba silos de información, ralentizaba la toma de decisiones y exigía un gran esfuerzo manual para la consolidación de métricas de calidad y avance.

Para mitigar este problema, el presente informe detalla el diseño y construcción de una solución tecnológica avanzada: un Bot Project Manager impulsado por Inteligencia Artificial (IA). En lugar de crear un simple webhook de notificaciones, se diseñó un ecosistema de orquestación inteligente basado en la plataforma \textbf{n8n}. Este sistema reacciona en tiempo real a los eventos del ciclo de vida del software, normaliza los datos, los enruta mediante un patrón \textit{Event-Driven} y emplea \textbf{Google Gemini 2.5 Pro} para el procesamiento analítico.

El resultado es un Project Manager Virtual autónomo capaz de realizar inspección técnica de \textit{Pull Requests}, detectar cuellos de botella mediante análisis de \textit{Aging WIP} (Work In Progress), consolidar reportes ejecutivos en PDF y gestionar un calendario PMO interno. Además, el sistema incorpora un riguroso patrón \textit{Human-in-the-Loop}, garantizando que toda decisión crítica sugerida por la IA (como bloqueos de despliegues o escalamientos) deba ser revisada y aprobada por un administrador humano mediante un dashboard seguro antes de su ejecución.

\section{OBJETIVOS}

\subsection{Objetivo General}
Diseñar, implementar y desplegar una arquitectura de orquestación inteligente y event-driven basada en n8n e Inteligencia Artificial Generativa que unifique la telemetría del proyecto PetSafe, automatice la inspección de código, optimice la gestión de riesgos y centralice la gobernanza del proyecto a través de canales de comunicación bidireccional.

\subsection{Objetivos Específicos}
\begin{enumerate}
    \item \textbf{Ingeniería de Flujos:} Desarrollar un sistema modular con más de 25 \textit{workflows} en n8n, segmentados en capas de Inteligencia, Infraestructura (Core), Comunicación, Monitoreo y Reportería (PMO).
    \item \textbf{Integración y Desacoplamiento:} Conectar las APIs REST de GitHub, Trello, Discord y Gemini, implementando \textit{Dead Letter Queues} (DLQ) e idempotencia para garantizar tolerancia a fallos.
    \item \textbf{Modelo de Datos Relacional y Vectorial:} Implementar una capa de persistencia en PostgreSQL con soporte para \texttt{pgvector}, estructurando eventos canónicos, auditorías de ejecución y almacenamiento auditable de decisiones de IA.
    \item \textbf{Agentes Cognitivos:} Configurar y programar agentes IA especializados, dotándolos de un \textit{System Prompt} contextualizado para inspecciones de código, gestión de prioridades y control de \textit{releases}.
    \item \textbf{Reporte Ejecutivo y Análisis de Deuda Técnica:} Automatizar la compilación de \textit{commits}, métricas de progreso y hallazgos en un informe PDF renderizado semanalmente mediante \textit{Gotenberg}.
    \item \textbf{Gobernanza y Seguridad:} Establecer protocolos de autorización (Dashboard protegido mediante tokens) y segregar el entorno mediante despliegues contenerizados en Docker con variables de entorno encriptadas.
\end{enumerate}

\section{DESARROLLO, ARQUITECTURA E IMPLEMENTACIÓN}

\subsection{Diseño de la Arquitectura Lógica}

La complejidad del proyecto exigió trascender el concepto de "automatización secuencial" y adoptar un paradigma \textbf{Micro-Modular Orientado a Eventos}. El sistema fue dividido en cinco pilares lógicos interconectados:

\begin{enumerate}
    \item \textbf{Capa de Comunicación (Edge Layer):} Encargada de recibir y enviar *payloads*. Está conformada por el puente Node.js (\texttt{bot-bridge}), webhooks dedicados para Discord, y *Triggers* nativos para GitHub y Trello. Funciona como punto de entrada y salida asíncrono.
    \item \textbf{Capa Core (Router \& Logger):} El "corazón" del flujo. Todo evento entrante se normaliza y se deposita en una tabla relacional \texttt{events\_inbox} garantizando la idempotencia frente a la red (evitando duplicidades de webhooks). Un enrutador maestro (\texttt{PM Orchestrator}) extrae de esta cola y clasifica la intención.
    \item \textbf{Capa de Inteligencia Artificial (Cognitive Agents):} Contiene los \textit{workflows} armados con la tecnología LangChain y Gemini 2.5 Pro. Cada sub-agente tiene un rol restringido y un *prompt* contextual que evita "alucinaciones" (ej., el agente de código solo revisa diffs, no responde sobre presupuestos).
    \item \textbf{Capa de Reportería (PMO \& Reporting):} Consolida grandes volúmenes de datos transaccionales de PostgreSQL, Trello y GitHub. Genera un modelo en JSON que luego es transformado mediante una plantilla HTML e inyectado a Gotenberg para emitir el Documento Ejecutivo PDF final.
    \item \textbf{Capa de Fiabilidad (Monitoring \& DLQ):} Un conjunto de flujos que vigila pasivamente el estado operativo. Detecta excepciones \textit{runtime} de n8n, ejecuta la recolección de mensajes fallidos (Dead Letter) y reporta anomalías de la propia infraestructura a los administradores.
\end{enumerate}

\subsection{Integración de Inteligencia Artificial (Gemini \& LangChain)}

El núcleo cognitivo reside en \textbf{Google Gemini 2.5 Pro}. La integración se logró evitando interacciones de una sola vía y estableciendo un mecanismo de "Agentes con Herramientas" (\textit{Tools/Functions}) mediante nodos nativos de LangChain en n8n. 

\textbf{Prompt Engineering Avanzado:} Se estructuraron *system prompts* robustos. Por ejemplo, el agente \texttt{Code Review Scout} recibe no solo el diff del commit, sino reglas de convención específicas del proyecto PetSafe (ej. "Priorizar validación de tokens JWT en backend, exigir tests para controladores").
Además, en la generación del Reporte Semanal, se emplea Gemini en modo de "Salida JSON Estructurada", obligando al LLM a devolver un esquema rígido que luego es inyectado sin rupturas a las plantillas HTML para su renderizado, solucionando problemas de inestabilidad de formatos.

\subsection{Implementación de la Gobernanza: Human-in-the-Loop}

Dado que un agente IA podría tomar decisiones erróneas basadas en datos parciales (ej. cancelar un despliegue crítico erróneamente), se desarrolló un \textbf{Dashboard de Aprobaciones PMO}.
\begin{enumerate}
    \item La IA evalúa una condición de alto riesgo y graba el resultado en \texttt{ai\_decisions} con el estado \texttt{pending\_approval}.
    \item El bot notifica en Discord a los desarrolladores Senior: "Alerta: Despliegue bloqueado por alto riesgo de regresión. Pendiente de aprobación manual".
    \item El administrador accede al \textit{Approval Hub} (una interfaz HTML servida dinámicamente mediante el nodo webhooks de n8n y autenticada vía \texttt{PM\_APPROVAL\_SECRET}).
    \item Tras analizar la sugerencia, el humano pulsa "Aprobar" o "Rechazar", evento que reanuda el pipeline bloqueado de n8n.
\end{enumerate}

\subsection{Ciberseguridad y Manejo de Entornos}

El despliegue priorizó la segregación y el encapsulamiento:
\begin{itemize}
    \item \textbf{Desacoplamiento de Secretos:} Ninguna clave API, contraseña o webhook se codificó en los nodos de n8n. Se implementó un flujo \texttt{System - Config Resolver} que inyecta los valores desde el entorno de Docker (\texttt{.env}).
    \item \textbf{Redes Aisladas:} Mediante \texttt{docker-compose.yml}, los contenedores de PostgreSQL, Redis y Gotenberg se ubicaron en una red interna que carece de puertos expuestos al host público, mitigando el riesgo de ataques externos a la base de datos.
    \item \textbf{Cifrado en Reposo:} Se estableció la variable \texttt{N8N\_ENCRYPTION\_KEY} para asegurar que todos los tokens Oauth o credenciales persistidas dentro de la base SQLite/Postgres de n8n estén cifradas criptográficamente.
    \item \textbf{Prevención de Inyecciones:} Se refactorizaron los nodos SQL para asegurar que toda interpolación utilice paso por parámetros de la base de datos subyacente (\texttt{queryReplacement}), anulando posibles ataques de inyección SQL originados en *payloads* maliciosos de GitHub.
\end{itemize}

\section{GUÍA OPERATIVA Y MANTENIMIENTO}

\subsection{Consumo de Interfaces para Usuarios Finales}

El punto de contacto humano principal es el servidor de Discord. El sistema soporta consultas naturalizadas (\textit{"¿Gemini, qué tan atrasados estamos hoy?"}) que disparan el motor LLM, y comandos estrictos para tareas deterministas:

\begin{itemize}
    \item \texttt{!estado} / \texttt{!progreso} / \texttt{!kpis}: Solicitud instantánea de series de tiempo y completitud.
    \item \texttt{!ci} / \texttt{!commits}: Recuperación inmediata de los artefactos de la API de GitHub.
    \item \texttt{!fase sync} / \texttt{!fase listar}: Conciliación forzada de columnas desde la API de Trello al motor Postgres local.
    \item \texttt{!raid}: Recuperación de la matriz de Riesgos, Suposiciones, Incidencias y Dependencias.
    \item \texttt{!reunion [tema]}: Invocación de un sub-agente de coordinación para emparejar calendarios del equipo y emitir convocatorias sin dependencia de Google Calendar.
\end{itemize}

\subsection{Evolutividad y Mantenimiento de Flujos}

El enfoque modular permite escalar el sistema sin impacto transversal:
\begin{enumerate}
    \item \textbf{Adición de Agentes:} Requiere crear un nuevo workflow independiente, conectarlo como sub-nodo (\textit{Execute Workflow Node}) en el \texttt{PM Orchestrator} y definir su condicional en el enrutador JSON.
    \item \textbf{Mantenimiento de Prompts:} Modificar la lógica cognitiva no requiere programación (despliegues de código base), bastando con ajustar los nodos de Prompt en el lienzo de n8n, lo cual aplica de forma instantánea.
    \item \textbf{Monitoreo de Salud:} El administrador de infraestructura debe inspeccionar esporádicamente la tabla \texttt{workflow\_runs\_audit} o teclear \texttt{!logs} en Discord para evaluar latencias, \textit{timeouts} y salud del caché en Redis.
\end{enumerate}

\section{CONCLUSIONES FINALES}

\begin{itemize}
    \item \textbf{Eficiencia Administrativa Cuantificable:} La automatización integral erradicó la consolidación manual de información, reduciendo a cero el tiempo empleado semanalmente por el gestor de proyecto en recolectar métricas y redactar informes de estado (los cuales ahora genera la IA en formato PDF al instante).
    \item \textbf{Robustez del Modelo Orientado a Eventos:} La adopción de las colas \texttt{inbox/outbox} y la \textit{Dead Letter Queue} ha garantizado que ninguna transacción, por pequeña que sea, se pierda por intermitencias de las APIs de Discord, Trello o GitHub, logrando alta disponibilidad.
    \item \textbf{Confianza Asistida por HITL:} Incorporar Inteligencia Artificial de frontera (Gemini 2.5 Pro) conlleva riesgos por "alucinación". La implementación del marco \textit{Human-in-the-Loop} eliminó el peligro sistémico, delegando el rol de la IA a consejero o auditor pasivo, y dejando el poder coercitivo en el usuario humano autenticado.
    \item \textbf{Madurez de Infraestructura Continua:} La contenedorización mediante Docker, sumada a la estricta parametrización SQL, encriptación de n8n y variables \texttt{.env}, dota al Bot de estándares comparables a un entorno Enterprise listo para la Nube, demostrando habilidades avanzadas de arquitectura de software.
\end{itemize}

\section{LÍNEAS FUTURAS Y RECOMENDACIONES TÉCNICAS}

Con base en la estabilidad actual, se proponen las siguientes rutas de evolución:
\begin{enumerate}
    \item \textbf{Cálculo de Métricas DORA Nativas:} Ampliar los analizadores de GitHub para procesar analíticamente el \textit{Lead Time for Changes}, \textit{Deployment Frequency}, \textit{MTTR} y \textit{Change Failure Rate}, incrustando dichas métricas de nivel devops en el Reporte Semanal.
    \item \textbf{RAG Semántico con pgvector:} Aprovechar la extensión \texttt{pgvector} ya instalada para crear embeddings de la documentación técnica antigua (ADRs, requisitos), permitiendo al bot responder con \textit{Retrieval-Augmented Generation} (RAG) citando la documentación original del proyecto PetSafe.
    \item \textbf{Vault Corporativo:} Migrar la resolución de secretos (\texttt{.env}) hacia un inyector como HashiCorp Vault, robusteciendo aún más la seguridad criptográfica del sistema.
    \item \textbf{Migración Parcial a SDK Discord.js:} Para mejorar la interactividad (UX), evolucionar el simple web-hook de respuesta por un servicio persistente \textit{WebSockets} utilizando \textit{Slash Commands} nativos y menús selectores enriquecidos en la interfaz de Discord.
\end{enumerate}
